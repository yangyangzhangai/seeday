## 2. 测试范围

### 2.1 纳入范围

| 模块编号 | 模块名称           | 测试重点                                               |
|----------|--------------------|--------------------------------------------------------|
| M01      | 登录 / 注册        | 邮箱、Apple Sign-In、Google 登录、会话保持             |
| M02      | 新手引导           | 7 步引导流程完整性、强制跳过、断点恢复                 |
| M03      | 聊天 / 记录        | 消息发送、活动卡/心情卡生成与操作、AI 批注、建议系统   |
| M04      | 成长 / 待办        | 待办 CRUD、重复任务、专注模式、瓶子目标                |
| M05      | 报告 / 日记        | 日报生成、日记书架、AI 日记（Plus）、Teaser（Free）   |
| M06      | 个人设置           | 语言切换、AI 人格、作息、AI 专属记忆、账号管理         |
| M07      | 通知 / 提醒        | 权限申请、主动提醒弹窗、晚间总结、作息修改后重排       |
| M08      | 会员 / 订阅        | 试用期、Free/Plus 功能门控、IAP 恢复购买               |
| M09      | 国际化（i18n）     | 三语界面完整性、语言切换联动、无硬编码文案            |
| M10      | iOS 移动端专项     | 安全区、键盘遮挡、触觉反馈、相机权限                   |

### 2.2 排除范围

- 后台 Vercel 函数配额压测
- App Store 上架审核流程（非本轮范围）

## 3. 优先级定义

| 优先级 | 定义                                               | 示例                         |
|--------|----------------------------------------------------|------------------------------|
| **P0** | 致命缺陷，直接阻断核心链路，必须上线前修复         | 无法登录、白屏崩溃、数据丢失 |
| **P1** | 主要功能异常，严重影响用户体验，必须上线前修复     | 日报无法生成、待办丢失       |
| **P2** | 次要功能缺陷，有替代路径，可在下个版本修复         | UI 文案错位、动画卡顿        |
| **P3** | 优化建议或轻微体验问题，不影响上线                 | 字体微调、间距不一致         |

---

## 4. 入口条件（Entry Criteria）

- [ ] 开发已完成所有 P0/P1 功能点
- [ ] `npm run lint:all` 全部通过（secrets / max-lines / docs-sync / tsc）
- [ ] `npm run build` 构建无报错
- [ ] iOS TestFlight 包已安装到测试设备
- [ ] 测试账号已准备（Free 账号 × 2 + Plus 账号 × 1）

---

## 5. 出口条件（Exit Criteria）

- [ ] P0 用例 100% 通过
- [ ] P1 用例 ≥ 95% 通过（允许 ≤ 1 条已记录缺陷挂起）
- [ ] P2 用例 ≥ 80% 通过
- [ ] 三语（zh / en / it）界面无明显错乱
- [ ] 所有已发现 P0/P1 缺陷已有修复记录或已关闭

## 7. 测试执行流程

```
准备测试账号
    ↓
执行 M01 登录/注册（P0 全通过才继续）
    ↓
执行 M02 新手引导
    ↓
并行执行 M03 聊天 + M04 成长
    ↓
执行 M05 报告/日记
    ↓
执行 M06 个人设置
    ↓
执行 M07 通知 + M08 会员
    ↓
专项：M09 国际化 + M10 iOS 移动端
    ↓
汇总缺陷，按优先级分类
    ↓
开发修复 → 回归测试（仅回归 P0/P1）
    ↓
达到出口条件 → 签字通过
```

## 8. 缺陷管理

| 状态     | 说明                                 |
|----------|--------------------------------------|
| Open     | 已发现，待开发确认                   |
| Confirmed| 开发已确认，排期修复                 |
| Fixed    | 已修复，待测试人员回归验证           |
| Closed   | 回归通过，关闭                       |
| Rejected | 非缺陷（预期行为 / 设计决策）        |
| Deferred | 延迟处理，下版本修复                 |
