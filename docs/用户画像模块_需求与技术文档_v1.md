## 1. 背景与目标

### 1.1 当前问题

AI 批注和建议系统存在机械感：每次建议像第一天认识用户。核心问题不是文案能力，而是缺少可持续记忆和人物理解。

当前缺失包括：

- 不知道用户节奏（起床/睡觉/吃饭、哪个时段更高效）
- 不知道用户偏好（喜欢什么、不喜欢什么）
- 不知道用户情绪模式（何时容易焦虑、何时更平稳）
- 不知道用户近期目标（备考、面试、减脂、学习等）
- 不知道历史重要事件（里程碑和高光/低谷时刻）

### 1.2 新定义：用户画像 = 人物理解 + 记忆系统

用户画像不再是问卷标签，而是四层并行系统：

| 层 | 内容 | 更新方式 |
|---|---|---|
| 用户声明层（manual） | 用户主动填写：作息、目标、可见纪念日等 | 手动更新 |
| 行为观测层（observed） | AI 从日记观察到的作息/情绪/效率模式 | 每周更新 |
| 动态信号层（decay） | 偏好、关系线索、情绪触发因素等 | 每周提取 + 月度衰减 |
| 事实记忆层（event memory） | 第一次体验/高光/低谷/关键里程碑 | 长期保留（不衰减） |

### 1.3 总体目标

- **短期（P0）**：引导采集最少必要信息 + 打通画像注入
- **中期（P1）**：周提取上线，形成“越用越懂你”
- **长期（P2）**：回忆召回系统（去年今日/上周今日/上月今日）稳定增强陪伴感

## 2. 讨论结论：吃饭提醒的去留

### 2.1 结论

吃饭提醒规则保留，不删除。改造重点是“个性化时段”，不再只依赖硬编码 `11-13 / 18-20`。

### 2.2 新规则

`isMealTime()` 同时参考两类数据：

1. 用户声明饭点（manual meal times）
2. 近期观测饭点（observed meal times）

默认优先声明值，但在实时建议可温和参考观测值。无画像时 fallback 到老逻辑。

```typescript
function isMealTime(hour: number, declared?: number[], observed?: number[]): boolean {
  const merged = declared?.length ? declared : observed;
  if (merged?.length) return merged.some((t) => Math.abs(hour - t) <= 1);
  return (hour >= 11 && hour <= 13) || (hour >= 18 && hour <= 20);
}
```

## 3. 讨论结论：纪念与记忆系统

### 3.1 双轨设计（已冻结）

#### A 类：可见纪念日（用户可管理）

- AI 判断为长期重要节点（如离职日、在一起纪念日）时，默认自动写入
- 用户可在“我的 > 画像 > 纪念日管理”中修改或删除

#### B 类：隐性事件记忆（仅 AI 可见）

- 包括高光时刻、低谷时刻、第一次体验等
- 不展示给用户列表，仅用于回复时召回
- 典型召回：去年今日、上周今日、上月今日

### 3.2 保留与衰减

- **B 类隐性事件记忆**：事实事件，长期保留，不做时间衰减
- **偏好/关系/状态型信号**：按月衰减置信度，长期不提及时降权

## 4. 功能需求（PRD）

### 4.1 功能模块全景

```
用户画像模块
├── F0. 长期画像开关（My 页）
│   └── 用户可选择是否开启长期画像能力
├── F1. 新手引导收集（Onboarding）
│   └── 采集最少必要信息（不做人格问卷）
├── F2. 我的页面编辑（Profile Edit）
│   └── 用户可随时更新声明信息
├── F3. AI 每周自动提取（Weekly Extraction）
│   └── 提取人物模式与动态信号
├── F4. 纪念与记忆系统（Memory）
│   └── A 类可见纪念日 + B 类隐性事件记忆
└── F5. 回忆召回（History Recall）
    └── 去年今日/上周今日/上月今日用于陪伴回复
```

### F0. 长期画像开关（新增）

**入口**：我的页面（Profile）

**开关文案建议**：`开启长期画像（让 AI 通过你的长期记录更懂你）`

**行为定义**：

- 开启：启动周提取、长期记忆写入、画像 prompt 注入、历史召回
- 关闭：整套长期画像链路完全停用（不提取、不写入、不注入、不召回）

### F1. 新手引导收集

**触发时机**：产品导览完成后触发，独立于功能导览。

**收集内容（不超过 5 步）**：

| 步骤 | 问题 | 字段 | 类型 |
|---|---|---|---|
| Step 1 | 你主要用这款 App 来做什么？ | `primaryUse` | 单选 |
| Step 2 | 你通常几点起床、睡觉、吃早/午/晚饭？ | `wakeTime`/`sleepTime`/`mealTimes` | 时间选择器 |
| Step 3 | 你最近最想推进的目标是什么？ | `currentGoal` | 单选 + 可自填 |
| Step 4 | 如果愿意，也可以写一个人生目标 | `lifeGoal` | 文本（可跳过） |
| Step 5 | 想让我记住的重要日期？（可跳过） | `anniversariesVisible` | 可选 1-3 个 |

**明确不收集**：年龄、伴侣状态、家庭关系等敏感信息。

### F2. 我的页面编辑

**入口**：Profile 页新增“我的画像”卡片。

**可编辑项**：

- 长期画像开关（F0）
- 作息声明（起床/睡觉/吃饭）
- 当前目标 / 人生目标
- 标签与自由文本
- A 类可见纪念日管理

### F3. AI 每周自动提取

**触发条件**：

- `longTermProfileEnabled = true`
- 距上次提取 >= 7 天
- 最近 7 天内满足最低记录量（默认 >= 5 条）

**提取维度（六层）**：

| 层 | 提取内容 | 示例 |
|---|---|---|
| 身份与角色线索 | 学习/工作状态、角色变化线索 | “近期以备考为主” |
| 作息与效率节奏 | 活跃时段、高效窗口、低能量窗口 | “9:00-11:00效率高” |
| 活动与偏好模式 | 喜欢/不喜欢做的事、饮食偏好 | “偏好跑步，少提聚会” |
| 情绪与调节模式 | 易波动时段、有效安抚方式 | “晚间易焦虑，散步有效” |
| 关系线索（被动） | 用户主动表露的人际状态 | “与朋友疏远感上升” |
| 阶段目标推断 | 近期计划、推进阻力 | “在准备面试，卡在复习节奏” |

**输出要求**：每条信号带 `confidence`、`evidenceCount`、`lastSeenAt`。

### F4. 纪念与记忆系统

**A 类可见纪念日**：

- 来源：用户手动 + AI 自动生成（长期重要节点）
- 可见可管理
- 触发：临近 <= 3 天、当天优先、24h 去重

**B 类隐性事件记忆**：

- 来源：AI 从日记提取首次体验/高光/低谷
- 不展示列表，仅 AI 内部使用
- 长期保留，不做衰减

### F5. 回忆召回

在无强新事件时，AI 可自然引用：

- 去年今日
- 上周今日
- 上月今日

用于增强陪伴感，不做系统推送。

## 5. 新手引导脚本（用户画像收集）

### 5.1 与现有 ONBOARDING_SPEC_v1.2 的关系

画像引导是独立“认识你”流程，在产品导览后启动，不插入原导览步骤。

```text
注册完成
  -> 产品导览（45s）
  -> tourCompleted = true
  -> 1s 后触发画像引导
  -> profileTourCompleted = true
```

### 5.2 整体结构（更新）

| 步骤 | 内容 | 操作方式 | 预估耗时 |
|---|---|---|---|
| P1 | 使用目的 | 4 选 1 | ~8s |
| P2 | 作息 + 早午晚饭点 | 时间选择 | ~12s |
| P3 | 近期目标 | 选项 + 自填 | ~8s |
| P4 | 人生目标（可选） | 文本输入 | ~8s |
| P5 | 纪念日（可跳过） | 可选添加 | ~8s |

### 5.3 脚本原则

- 每步可跳过
- 不直接询问“你是什么性格”
- 不收集年龄/伴侣/家庭关系
- 目标字段与待办“人生目标管理”双向同步

### 5.4 Store 扩展（useOnboardingStore）

## 6. 技术方案

### 6.1 存储策略

首期继续使用 Supabase Auth `user_metadata`，新增 `user_profile_v2`。

同时在 `preferences` 中增加总开关：`longTermProfileEnabled: boolean`。

### 6.2 双来源作息策略（已冻结）

作息不覆盖，只并存：

- `manualSchedule`：用户声明值
- `observedSchedule`：AI 近期观测值

在 prompt 中同时注入两者，告诉模型“声明优先，观测用于温和适配”。

### 6.3 全链路门控（关键）

统一门控函数：`isLongTermProfileEnabled(userPreferences)`。

以下入口必须门控：

1. 周提取触发入口
2. A/B 记忆写入入口
3. prompt 画像注入入口
4. 历史召回入口

若开关关闭，统一短路返回空（不读、不写、不调用额外 LLM 画像能力）。

### 6.4 `buildUserProfileSnapshot()`

位置：`src/lib/buildUserProfileSnapshot.ts`（新增）

```typescript
interface UserProfileSnapshot {
  text: string;
  mealTimesForSuggestion?: number[];
  visibleUpcomingAnniversaries?: VisibleAnniversary[];
  hiddenRecallMoments?: HiddenMoment[];
}
```

职责：

1. 合并 manual + observed + aiSignals
2. 输出可读文本（英文，用于 LLM）
3. 计算临近纪念日
4. 计算可用历史召回素材

### 6.5 `isMealTime()` 改造

位置：`src/lib/suggestionDetector.ts`

签名升级为接收声明值与观测值，fallback 保持老逻辑。

### 6.6 `api/extract-profile.ts`

```text
POST /api/extract-profile
Body: { userId, recentMessages[] }
Response: {
  observedSchedule,
  dynamicSignals,
  visibleAnniversaryCandidates,
  hiddenMomentCandidates
}
```

输出必须走严格 JSON schema 校验，不合格直接丢弃，不影响主链路。

### 6.7 Prompt 注入

修改：`src/server/annotation-prompts.ts`

注入顺序建议：

1. 用户声明（manual）
2. 行为观测（observed）
3. 动态信号（带置信度）
4. 临近纪念日 / 历史召回候选

### 6.8 衰减策略

| 类型 | 策略 |
|---|---|
| 事实事件记忆（B 类） | 长期保留，不衰减 |
| 偏好/关系/状态信号 | 30 天降权，60 天弱化，90 天默认不主动触发 |
| 作息/效率/情绪节奏 | 每周重算，滚动窗口更新 |

### 6.9 隐私边界

- 不主动收集年龄、伴侣、家庭关系
- 仅当用户主动表达时，允许 AI 记录关系线索
- 关系线索仅后台用于陪伴回复，不前台展示

## 7. 数据结构设计

### 7.1 完整类型定义（v2）

```typescript
export type PrimaryUse =
  | 'life_record'
  | 'organize_thoughts'
  | 'emotion_management'
  | 'habit_building';

export type LifeStage = 'student' | 'employed' | 'freelance' | 'other';

export interface VisibleAnniversary {
  id: string;
  label: string;
  date: string; // MM-DD or YYYY-MM-DD
  repeating: boolean;
  source: 'user' | 'ai_auto';
  createdAt: string;
}

export interface HiddenMoment {
  id: string;
  kind: 'first_time' | 'highlight' | 'lowlight' | 'milestone';
  title: string;
  date: string; // YYYY-MM-DD
  summary: string;
  sourceMessageIds: string[];
  createdAt: string;
}

export interface ConfidenceSignal<T = string> {
  value: T;
  confidence: number;
  evidenceCount: number;
  lastSeenAt: string;
}

export interface UserProfileManual {
  primaryUse?: PrimaryUse;
  lifeStage?: LifeStage;
  wakeTime?: string;
  sleepTime?: string;
  mealTimes?: number[];
  currentGoal?: string;
  lifeGoal?: string;
  tags?: string[];
  freeText?: string;
}

export interface UserProfileObserved {
  wakeTime?: ConfidenceSignal<string>;
  sleepTime?: ConfidenceSignal<string>;
  mealTimes?: ConfidenceSignal<number[]>;
  activeWindows?: ConfidenceSignal<string[]>;
  moodByTimeBand?: ConfidenceSignal<Record<string, string>>;
  efficiencyByTimeBand?: ConfidenceSignal<Record<string, string>>;
}

export interface UserProfileDynamicSignals {
  preferences?: ConfidenceSignal<string[]>;
  dislikes?: ConfidenceSignal<string[]>;
  copingPatterns?: ConfidenceSignal<string[]>;
  relationshipSignals?: ConfidenceSignal<string[]>;
  currentFocusInference?: ConfidenceSignal<string[]>;
}

export interface UserProfileV2 {
  manual: UserProfileManual;
  observed?: UserProfileObserved;
  dynamicSignals?: UserProfileDynamicSignals;
  anniversariesVisible?: VisibleAnniversary[];
  hiddenMoments?: HiddenMoment[];
  onboardingCompleted?: boolean;
  lastExtractedAt?: string;
  createdAt: string;
  updatedAt: string;
}
```

## 9. 验收标准

### 开关与门控

- [ ] 开关在我的页面可见并可切换
- [ ] 开关关闭时，长期画像链路全部停用（0 提取 / 0 写入 / 0 注入 / 0 召回）
- [ ] 开关开启后，链路恢复且可正常产出画像

### 新手引导

- [ ] 新用户触发，老用户不重复触发
- [ ] 不询问性格类型，不收集年龄/伴侣/家庭关系
- [ ] 目标字段与作息饭点写入成功

### 作息与建议

- [ ] 支持 manual 与 observed 双来源并存，不覆盖
- [ ] 吃饭提醒命中用户饭点（±1 小时）
- [ ] 无画像时 fallback 到 11-13 / 18-20

### 纪念与回忆

- [ ] A 类可见纪念日可被用户管理
- [ ] B 类隐性事件记忆不展示列表但可被 AI 召回
- [ ] 去年今日/上周今日/上月今日可用于自然回复

### 周提取与衰减

- [ ] 每 7 天最多一次提取
- [ ] 提取失败不影响主链路
- [ ] 动态信号衰减符合 30/60/90 策略
- [ ] 事实事件记忆不衰减

## 10. 隐私与开关策略

### 10.1 不主动采集边界

- 不主动收集年龄、伴侣关系、家庭关系
- 用户主动表露时，允许后台记录关系线索（仅陪伴语境使用）

### 10.2 可见性边界

- A 类纪念日：用户可见、可管理
- B 类隐性事件记忆：用户不可见，仅 AI 使用

### 10.3 关闭开关后的行为

- 立即停用长期画像链路
- 历史数据默认冷存不使用
- 后续可在设置提供“清除长期画像数据”入口（单独确认后删除）
